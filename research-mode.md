# 🔬 研究模式（用户主动选择，不自动触发）

## 定位
基于多视角研究方法论（STORM）的内置深度研究模式。**默认不启动**，仅在用户确认后进入。

## 触发逻辑
```
检测到用户请求包含以下信号时：
  核心动词 ⊆ {研究, 分析, 对比, 探索, 了解, 评估, 调查}
  且 研究对象 ⊆ {领域, 技术, 概念, 趋势, 现象, 方法}
→ 不自动执行，弹出一个选项：
  "这个任务需要我开启多视角研究模式吗？
   当前模式会消耗更多 Token（约 5000-7000 token），
   但能产出含多视角扫描、矛盾地图、综合简报和自评的完整研究结果。
   说'开启研究模式'就开始，不然我直接用标准方式处理。"
用户确认后才进入研究模式。
```

## 研究模式工作流（4 步）

| 步骤 | 动作 | 输出 | Token 预估 |
|:----|:----|:----|:----------|
| 1 | 多视角扫描：模拟 5 类专家视角（实践者/学者/怀疑者/经济观察者/历史视角） | 5 视角卡片 | 1500-2000 |
| 2 | 矛盾地图：找出视角间的冲突点、共识区和盲区 | 矛盾列表 + 盲区标注 | 1000-1500 |
| 3 | 综合研究简报：CEO 级摘要 + 5 条核心发现（含可靠性排序）+ 隐藏关联 + 行动建议 + 前沿问题 | 研究报告 | 1500-2000 |
| 4 | 自评与同行评审：置信度评分、偏差检查、缺失视角 | 评审意见 + 整体评级 | 1000-1500 |

## 与普通推荐的区别

| | 普通模式 | 研究模式 |
|:--|:--------|:--------|
| 触发方式 | 自动路由 | 用户主动确认后进入 |
| Token 消耗 | <500 token | 5000-7000 token |
| 产出物 | Skill 推荐 + 对比 | 多视角研究简报 |
| 适用场景 | 日常任务 | 深度研究、决策前调研、文章写作前准备 |

## 退出机制
用户随时可以说「退出研究模式」或「够了」→ 立即停止当前流程，返回到普通推荐模式。
