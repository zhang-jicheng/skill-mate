## 🧪 测试 Prompts

### Test 1: 典型场景 — 找文献

```
prompt: "帮我找几篇关于中医功法对老年人平衡能力影响的文献"
期望行为:
  Step 2: 意图分析 → 核心动词"找"、研究对象"文献"、域"academic"
  Step 3: 匹配 → 推荐 "文献检索" 或 "nature-academic-search"
  Step 4: 推荐呈现 → 推荐理由 + 同类对比（已安装的学术相关 Skill）
```

### Test 2: 模糊请求 + 无匹配

```
prompt: "帮我整理一下"
期望行为:
  Step 2: 意图分析 → 核心动词"整理"、但对象/产出物不明确
  → 🟡 引导追问：具体想整理什么？文件/笔记/数据/思路？
  → 2 轮追问仍模糊 → 降级处理
```

### Test 3: 跨域复杂请求

```
prompt: "我手头有一批预实验数据，想分析一下差异，再画个图展示结果"
期望行为:
  Step 2: 意图分析 → 核心动词"分析"+"画"、域"data"+"writing"、跨域=true
  Step 3: 匹配 → 推荐统计分析的 Skill + 绘图的 Skill
  Step 4: 推荐呈现 → 分步推荐："先用 A 分析数据，再用 B 画图"
  Step 5: 执行 A → 结果分析 → 推荐 B → 执行 B
```

### Test 4: 无 Skill 可用 + 降级

```
prompt: "今天天气怎么样"
期望行为:
  Step 2: 意图分析 → 核心动词"查"、研究对象"天气"、无匹配 Skill
  → 告知无匹配 Skill + 用自己的能力查天气（如果能查）
  → 或告知"目前没有查天气的 Skill，不过我可以直接告诉你我知道的"
```

### Test 5: 用户主动选择

```
prompt: "用文献检索帮我找平衡能力的文章"
期望行为:
  → 用户已经指定了 Skill 名称 → 跳过 Step 3-4
  → 直接 Step 5: 加载"文献检索" → 传递请求 → 执行
```

### Test 6: 科研多阶段请求 → 推编排器

```
prompt: "帮我设计一个关于CRISPR基因编辑的实验方案，然后分析数据写论文"
期望行为:
  Step 2: 意图分析 → 核心动词"设计+分析+写"、研究阶段[实验+统计+写作] ≥2
  → 域专属路由 → 首选推 research-orchestrate（编排器，覆盖全流程）
  → 备选：实验设计（子任务）+ 统计分析（子任务）+ 学术写作（子任务）
  → 理由：全流程有编排器，比逐个用工具省心
```

### Test 7: 认知困惑 → 先推认知工具

```
prompt: "我想不通为什么我的产品卖不动"
期望行为:
  Step 1: 意图分析 → 核心动词"想不通"、无具体任务对象、表达困惑
  → 认知路由优先 → 首选推 ljg-think（如果再钻一层：定价/人群/模式）
  → 如果 ljg-think 未安装 → 降级为 skill-mate 直接认知对话
     "我们来一层一层拆——你觉得卖不动的第一个信号是什么？"
```

## 🔬 A/B 测试设计

skill-mate 的推荐质量需要可量化验证。以下定义了 "with skill-mate" vs "baseline（无 skill-mate）" 的对照实验框架。

### 实验设计总览

| 变量 | With skill-mate | Baseline（无 skill-mate） |
|------|----------------|--------------------------|
| 触发方式 | skill-mate 自动匹配并推荐 | 用户自行翻 Skill 列表或直接让 AI 回答 |
| 推荐精度 | 输出 Top 1-3 + 理由 + 同类对比 | 无结构化推荐 |
| 用户决策成本 | 说"就用X"即可执行 | 需自行研究哪些 Skill 能用 |
| 追踪链路 | 意图分析→匹配→推荐→确认→执行→下一步 | 无追踪链路 |
| 后续衔接 | 执行完主动推荐下一步 Skill | 结束即结束，无后续 |

### 评测指标

| 指标 | 定义 | 采集方式 |
|------|------|---------|
| **首推命中率** | 用户选首推的比例（应 ≥ 70%） | 统计 Step 5 确认的是否为首推 |
| **推荐耗时** | 从用户输入到给出推荐的时间（秒） | 手动计时或日志 |
| **用户决策轮次** | 用户说"换一个"的次数（应 ≤ 1） | 统计 Step 5 分支走向 |
| **最终满意度** | 用户是否完成目标，匿名评分 1-5（应 ≥ 4） | 执行完成后询问 |
| **对比提升率** | with vs baseline 的首推命中率差值 | A/B 对照计算 |

### 对照实验流程

```
1. 准备 10 个标准化测试 prompt（覆盖 5 类场景 × 2）
2. 分别在 "with skill-mate" 和 "baseline" 两种模式下运行
3. 记录每个 prompt 的推荐结果到 test/results.tsv
4. 独立评委（另一 LLM 或无 skill-mate 的会话）按评分卡打分
5. 计算两组差异 → 输出对比报告
```

### 测试场景覆盖

| 场景类别 | Test # | 示例 prompt |
|---------|--------|------------|
| 明确学术任务 | 1A, 1B | "帮我找几篇平衡能力训练的文献" |
| 模糊日常请求 | 2A, 2B | "帮我整理一下这些笔记" |
| 跨域复杂任务 | 3A, 3B | "分析这批数据再画个图展示" |
| 明确指定 Skill | 4A, 4B | "用文献检索帮我找文章" |
| 无匹配 Skill | 5A, 5B | "今天天气怎么样" |

## 📊 评委评分

每次 A/B 测试或实际运行后，由独立评委（不同模型，或不含 skill-mate 的独立会话）按以下评分卡对每次推荐打分。

### 评分卡（满分 25 分，转换为百分制）

| 维度 | 权重 | 5 分 | 3 分 | 1 分 |
|------|:---:|------|------|------|
| **推荐准确度** | ×6 | 首推完全匹配用户意图 | 备选匹配，首推有偏差 | 3 推荐均不匹配 |
| **理由说服力** | ×5 | 具体场景化，说人话 | 有理由但偏模板 | 无理由或纯技术术语 |
| **同类透明度** | ×4 | 列 ≥2 同类 + 区别说明 | 列同类但无区别 | 未提及同类 |
| **确认成本** | ×5 | 一句话可确认执行 | 需多轮才能确认 | 用户需自行研究 |
| **后续衔接** | ×5 | 主动推荐下一步 Skill | 被动等待指令 | 执行完即结束 |

**总分 = Σ(维度得分 × 权重) / 25 × 100**

| 得分区间 | 等级 | 说明 |
|---------|:---:|------|
| 90-100 | 🟢 优秀 | 推荐流畅，用户零摩擦 |
| 75-89 | 🟡 良好 | 有小瑕疵但不影响使用 |
| 60-74 | 🟠 需改进 | 推荐质量不稳定 |
| <60 | 🔴 不合格 | 建议关停修 bug |
